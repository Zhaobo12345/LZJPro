# 小程序端分享功能设计文档

## 1. 功能概述

为家装平台小程序增加分享给朋友功能，允许项目成员将项目信息分享给非项目成员查看，并支持非项目成员申请加入项目。

## 2. 设计目标

- **分享便捷性**：在现有页面的分享入口基础上，提供完整的分享流程
- **信息安全**：确保敏感信息（如详细地址、联系方式、合同金额）在分享时得到保护
- **权限控制**：非项目成员只能查看受限信息，不能进行编辑操作
- **加入机制**：非项目成员可以申请加入非完工状态的项目

## 3. 页面设计

### 3.1 分享卡片接收页 (`prototype-share-receive.html`)

**页面结构**：
- 顶部导航栏：返回按钮、标题、关闭按钮
- 项目基本信息卡片：项目名称、状态、地址（省市区可见，详细地址加密）
- 项目信息卡片：客户姓名、面积、房型、风格（联系方式加密）
- 合同概览卡片：合同状态、金额（加密）、无权限提示
- 项目成员卡片：成员头像列表（只读）
- 项目架构卡片：架构层级（只读）
- 申请加入按钮：仅对非完工状态项目显示
- 登录提示：引导用户登录查看更多
- 分享提示：提示用户正在查看分享的项目

**权限控制**：
| 内容 | 可见性 |
|------|--------|
| 项目名称 | ✅ 可见 |
| 项目地址（省市区） | ✅ 可见 |
| 项目地址（详细地址） | 🔒 加密显示 |
| 客户姓名 | ✅ 可见 |
| 客户联系方式 | 🔒 加密显示 |
| 合同概览 | ✅ 可见（无金额） |
| 合同金额 | 🔒 不可见 |
| 合同正文 | 🔒 不可见 |
| 任务详情 | ✅ 可见（只读） |
| 项目成员 | ✅ 可见（只读） |
| 项目架构 | ✅ 可见（只读） |
| 创建/编辑操作 | ❌ 隐藏 |
| 邀请/删除成员 | ❌ 隐藏 |
| 申请加入项目 | ✅ 可申请（非完工状态） |

### 3.2 申请加入项目页 (`prototype-join-request.html`)

**页面结构**：
- 顶部导航栏：返回按钮、标题、关闭按钮
- 项目信息卡片：项目名称、项目经理
- 申请理由：文本域（必填）
- 角色选择：水电工、泥瓦工、木工、油漆工、其他（单选）
- 联系方式：手机号输入框（必填）
- 提交按钮：提交申请
- 提示信息：申请流程说明

**功能流程**：
1. 用户填写申请理由、选择角色、输入联系方式
2. 点击提交按钮进行表单验证
3. 验证通过后显示成功提示
4. 自动跳转回分享接收页

## 4. 分享入口

**现有分享入口**：
- 项目详情页 (`prototype-project-detail.html`)
- 任务详情页 (`prototype-task-detail.html`)
- 合同详情页 (`prototype-contract-detail.html`)
- 活动列表页 (`prototype-activity-list.html`)
- 成员档案页 (`prototype-member-profile.html`)
- 临时任务详情页 (`prototype-temp-task-detail.html`)

**分享行为**：
- 点击分享按钮后，跳转到 `prototype-share-receive.html` 页面
- 模拟分享功能，展示分享后的查看效果

## 5. 技术实现

### 5.1 页面创建
- `prototype-share-receive.html`：分享卡片接收页
- `prototype-join-request.html`：申请加入项目页

### 5.2 功能实现
- 更新现有页面的 `shareToFriend` 函数，跳转到分享接收页
- 实现分享接收页的权限控制逻辑
- 实现申请加入项目的表单验证和提交逻辑

### 5.3 数据传递
- 实际项目中通过 URL 参数传递项目 ID
- 分享接收页根据项目 ID 加载项目数据
- 申请加入页根据项目 ID 提交申请

## 6. 分享链接有效期

- 分享链接永久有效
- 除非项目被删除或分享人主动取消分享

## 7. 登录要求

- 被分享人无需登录即可查看分享内容
- 登录后可查看更多详细信息（如完整地址、联系方式等）

## 8. 加入项目规则

- 项目非已完工状态时，非项目成员可以申请加入
- 申请需填写理由、选择角色、提供联系方式
- 申请提交后，项目经理收到通知并进行审批
- 审批通过后，申请人成为项目成员

## 9. 安全考虑

- 敏感信息在分享时进行加密处理
- 非项目成员只能查看受限信息
- 所有操作都有适当的权限验证
- 分享链接使用安全的参数传递方式
