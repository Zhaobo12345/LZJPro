# 小程序端分享功能设计文档

## 1. 功能概述

为家装平台小程序增加分享给朋友功能，允许项目成员将项目信息分享给非项目成员查看，并支持非项目成员申请加入项目。

## 2. 设计目标

- **分享便捷性**：在现有页面的分享入口基础上，提供完整的分享流程
- **信息安全**：确保敏感信息（如详细地址、联系方式、合同金额）在分享时得到保护
- **权限控制**：非项目成员只能查看受限信息，不能进行编辑操作
- **加入机制**：非项目成员可以申请加入非完工状态的项目

## 3. 页面结构

### 3.1 分享功能导航页 (`share-navigation.html`)

**功能说明**：集中展示所有分享功能相关页面，方便测试和导航。

**页面内容**：
- 项目详情分享
- 项目动态分享
- 项目任务分享
- 项目合同列表
- 项目合同详情
- 申请加入页

### 3.2 项目详情分享接收页 (`prototype-share-receive.html`)

**页面结构**：
- 状态栏
- 导航栏：返回按钮、标题"分享的项目"、关闭按钮
- 分享提示横幅：提示用户正在查看分享的项目
- 固定顶部的快捷导航：项目信息、项目动态、项目任务、项目合同
- 项目基本信息卡片：项目名称、状态、地址（省市区可见，详细地址加密）
- 客户信息卡片：客户姓名、面积、房型、风格、联系方式（加密）
- 合同概览卡片：合同总数、总进度、合同列表
- 项目成员卡片：成员头像列表（只读）
- 项目架构卡片：架构层级（只读）
- 申请加入按钮：仅对非完工状态项目显示
- 登录提示：引导用户登录查看更多

**权限控制**：
| 内容 | 可见性 |
|------|--------|
| 项目名称 | ✅ 可见 |
| 项目地址（省市区） | ✅ 可见 |
| 项目地址（详细地址） | 🔒 加密显示 |
| 客户姓名 | ✅ 可见 |
| 客户联系方式 | 🔒 加密显示 |
| 合同概览 | ✅ 可见（无金额） |
| 合同金额 | 🔒 不可见 |
| 合同正文 | 🔒 不可见 |
| 任务详情 | ✅ 可见（只读） |
| 项目成员 | ✅ 可见（只读） |
| 项目架构 | ✅ 可见（只读） |
| 创建/编辑操作 | ❌ 隐藏 |
| 邀请/删除成员 | ❌ 隐藏 |
| 申请加入项目 | ✅ 可申请（非完工状态） |

### 3.3 项目动态分享接收页 (`prototype-share-receive-activity.html`)

**页面结构**：
- 状态栏
- 导航栏：返回按钮、标题"分享的项目动态"、关闭按钮
- 分享提示横幅
- 项目基本信息卡片
- 动态列表：
  - 按时间倒序展示最新活动
  - 每条动态包含：时间、类型、内容、相关人员
  - 支持展开/收起更多历史动态
  - 不同类型活动使用不同颜色圆点标识
- 申请加入按钮
- 登录提示

**权限控制**：
- 仅显示动态内容，无发布/编辑权限
- 敏感信息适当加密
- 非完工状态项目可申请加入

### 3.4 项目任务分享接收页 (`prototype-share-receive-task.html`)

**页面结构**：
- 状态栏
- 导航栏：返回按钮、标题"分享的项目任务"、关闭按钮

- 项目基本信息卡片
- 任务列表：
  - 按合同分组展示任务
  - 每个任务显示：名称、状态、执行人、截止日期

- 任务详情弹窗：
  - 任务名称、状态、执行人、确认人
  - 执行标准、确认标准、担责标准
  - 执行记录（只读，敏感信息加密）
  - 确认记录（只读）
- 申请加入按钮
- 登录提示

**权限控制**：
- 仅显示任务信息，无操作权限
- 敏感信息适当加密
- 非完工状态项目可申请加入

### 3.6 项目合同详情分享接收页 (`prototype-share-receive-contract-detail.html`)

**页面结构**：
- 状态栏
- 导航栏：返回按钮、标题"合同详情"、关闭按钮
- 分享提示横幅
- 项目基本信息卡片
- 合同信息卡片：
  - 合同名称、类型、状态
  - 甲乙方名称（适当加密）
  - 合同金额（加密）
  - 工期、签约日期
- 合同正文预览：
  - 显示合同正文摘要
  - 敏感信息加密
  - 无权限查看完整正文
- 阶段任务概览：
  - 按阶段展示任务
  - 每个阶段显示完成情况
  - 点击任务可查看任务详情弹窗
- 任务详情弹窗：
  - 任务名称、状态、执行人、确认人
  - 执行标准、确认标准、担责标准
  - 执行记录（只读）
  - 确认记录（只读）
- 申请加入按钮
- 登录提示

**权限控制**：
- 仅显示合同概览，无编辑权限
- 敏感信息（金额、详细内容）加密
- 非完工状态项目可申请加入

### 3.7 申请加入项目页 (`prototype-join-request.html`)

**页面结构**：
- 状态栏
- 导航栏：返回按钮、标题"申请加入"、关闭按钮
- 项目信息卡片：项目名称、项目经理
- 申请理由：文本域（必填）
- 角色选择：水电工、泥瓦工、木工、油漆工、其他（单选）
- 联系方式：手机号输入框（必填）
- 提交按钮：提交申请
- 提示信息：申请流程说明

**功能流程**：
1. 用户填写申请理由、选择角色、输入联系方式
2. 点击提交按钮进行表单验证
3. 验证通过后显示成功提示
4. 自动跳转回分享接收页

## 4. 分享入口

**现有分享入口**：
- 项目详情页 (`prototype-project-detail.html`)
- 任务详情页 (`prototype-task-detail.html`)
- 合同详情页 (`prototype-contract-detail.html`)
- 活动列表页 (`prototype-activity-list.html`)
- 成员档案页 (`prototype-member-profile.html`)

**分享行为**：
- 点击分享按钮后，显示分享选项提示
- 选择分享方式后跳转到对应的分享接收页

## 5. 技术实现

### 5.1 页面文件
- `share-navigation.html`：分享功能导航页
- `prototype-share-receive.html`：项目详情分享接收页
- `prototype-share-receive-activity.html`：项目动态分享接收页
- `prototype-share-receive-task.html`：项目任务分享接收页
- `prototype-share-receive-contract-list.html`：项目合同列表分享接收页
- `prototype-share-receive-contract-detail.html`：项目合同详情分享接收页
- `prototype-join-request.html`：申请加入项目页

### 5.2 功能实现
- 更新现有页面的 `shareToFriend` 函数，显示分享选项提示
- 实现各分享接收页的权限控制逻辑
- 实现申请加入项目的表单验证和提交逻辑
- 任务详情弹窗在手机框内正确显示

### 5.3 数据传递
- 实际项目中通过 URL 参数传递：
  - `project_id`：项目 ID
  - `share_type`：分享类型（detail/activity/task/contract）
  - `item_id`：具体项目 ID（如任务 ID、合同 ID）
- 分享接收页根据参数加载对应数据

### 5.4 弹窗定位
- 任务详情弹窗使用 `position: absolute` 相对于手机框定位
- 弹窗 HTML 结构必须位于手机框内部
- 尺寸设置确保弹窗在手机框内显示

## 6. 分享链接有效期

- 分享链接永久有效
- 除非项目被删除或分享人主动取消分享

## 7. 登录要求

- 被分享人无需登录即可查看分享内容
- 登录后可查看更多详细信息（如完整地址、联系方式等）

## 8. 加入项目规则

- 项目非已完工状态时，非项目成员可以申请加入
- 申请需填写理由、选择角色、提供联系方式
- 申请提交后，项目经理收到通知并进行审批
- 审批通过后，申请人成为项目成员

## 9. 安全考虑

- 敏感信息在分享时进行加密处理
- 非项目成员只能查看受限信息
- 所有操作都有适当的权限验证
- 分享链接使用安全的参数传递方式
