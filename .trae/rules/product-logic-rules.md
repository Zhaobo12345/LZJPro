# 家装平台小程序 - 产品逻辑规则

## 生效方式
始终生效

---

## 一、项目层级架构规则

### 1.1 项目层级定义
平台支持最多4层项目分解结构：

| 层级 | 名称 | 说明 | 代码标识 |
|------|------|------|----------|
| Level 1 | 项目部 | 最顶层项目组织 | level-0 |
| Level 2 | 工作组 | 专业分工组（水电/泥瓦/木工/油漆） | level-1 |
| Level 3 | 施工组 | 工作组下的具体施工小组 | level-2 |
| Level 4 | 任务包 | 最小执行单元 | 任务项 |

### 1.2 节点操作权限规则
- 只有项目经理角色才能创建/删除架构节点
- 创建施工组前，必须先确认所属工作组已创建合同
- 删除工作组时，会级联删除其下所有施工组和相关合同

### 1.3 合同与架构绑定规则
- 每个工作组（Level 2）最多关联一个合同
- 创建施工组前，必须先为该工作组创建合同
- 合同状态包括：拟定中、待签署、执行中、已完成

### 1.4 项目树交互规则
- 点击节点可展开/收起子节点
- 支持添加工作组、添加施工组、邀请成员、删除等操作
- 节点展开/收起有动画过渡效果

## 二、合同管理规则

### 2.1 合同状态定义

| 状态 | 显示文字 | 说明 |
|------|----------|------|
| draft | 拟定中 | 合同已创建，请前往电脑端完善 |
| confirming_sender | 待对方确认 | 已发起确认申请，等待对方确认 |
| confirming_receiver | 待我方确认 | 对方已确认，请我方最终确认 |
| confirmed | 已确认 | 双方已确认，请上传签约文件 |
| signing | 待确认签约 | 对方已上传签约文件，请确认 |
| signing_wait | 待对方确认签约 | 签约文件已上传，等待对方确认 |
| signed | 已签约 | 合同已正式生效，可发起变更 |
| changing | 变更中 | 变更申请已发起，等待对方确认 |
| change_confirming | 待确认变更 | 对方发起变更，请确认或驳回 |

### 2.2 合同状态流转规则

| 当前状态 | 允许操作 | 下一状态 |
|----------|----------|----------|
| 拟定中 | 提交确认 | 待对方确认 |
| 待对方确认 | 撤回确认 | 拟定中 |
| 待我方确认 | 确认合同 | 已确认 |
| 待我方确认 | 驳回修改 | 拟定中 |
| 已确认 | 上传签约文件 | 待对方确认签约 |
| 待对方确认签约 | 撤回签约文件 | 已确认 |
| 待确认签约 | 确认签约文件 | 已签约 |
| 待确认签约 | 驳回 | 已确认 |
| 已签约 | 发起变更 | 变更中 |
| 变更中 | 确认变更 | 已签约 |
| 变更中 | 撤回变更 | 已签约 |

### 2.3 合同签约双方状态同步规则

| 合同状态 | 甲方状态 | 乙方状态 |
|----------|----------|----------|
| 拟定中 | 编辑中 | 待确认 |
| 待对方确认 | 已确认 | 确认中 |
| 待我方确认 | 已确认 | 确认中 |
| 已确认 | 已确认 | 已确认 |
| 待对方确认签约 | 已上传签约 | 待确认签约 |
| 待确认签约 | 待确认签约 | 已上传签约 |
| 已签约 | 已签约 | 已签约 |

### 2.4 合同变更规则
- 发起变更后，阶段任务暂停流转
- 变更需要双方确认后生效
- 变更生效后生成新的版本记录
- 合同正文修改需在电脑端操作（移动端仅查看）

## 三、任务管理规则

### 3.1 任务状态定义

| 状态 | 说明 | 按钮显示 |
|------|------|----------|
| pending | 待开始 | 开始执行 |
| confirming | 确认中 | 驳回 / 确认通过 |
| completed | 已完成 | 查看记录 / 查看评价 |

### 3.2 任务执行流程
- 执行人接收任务后，状态变为"待开始"
- 执行人提交执行记录后，状态变为"确认中"
- 所有确认人确认后，状态变为"已完成"
- 任一确认人驳回，任务退回执行人重新执行

### 3.3 确认人机制
- 每个任务可设置最多5个确认人
- 确认人可给出评分（1-5星）
- 确认进度显示格式：`确认进度：已确认数/总确认人数`
- 确认时可上传图片、视频、音频

### 3.4 三层标准规则
每个任务包含三个维度的标准：

| 标准类型 | 用途 | 展示位置 |
|----------|------|----------|
| 执行标准 | 指导执行人如何完成任务 | 可折叠卡片 |
| 确认标准 | 指导确认人如何验收 | 可折叠卡片 |
| 担责标准 | 明确责任归属 | 可折叠卡片 |

## 四、临时任务规则

### 4.1 临时任务生命周期
待接收 → 待开始 → 确认中 → 已完成
↓
已拒绝


### 4.2 派发与接收机制
- 任务派发后，执行人可选择"接受"或"拒绝"
- 拒绝时需要填写拒绝原因
- 支持批量接受多个待处理任务

### 4.3 任务状态颜色标识

| 状态 | 颜色 | 代码类名 |
|------|------|----------|
| 待接收 | 橙色 | pending |
| 待开始 | 橙色 | pending |
| 确认中 | 蓝色 | confirming |
| 已完成 | 绿色 | completed |
| 已拒绝 | 红色 | rejected |

## 五、项目详情页规则

### 5.1 快捷入口
- 四个快捷入口：项目信息、架构、成员、资料
- 支持收起/展开快捷入口（滚动时自动收起）
- 收起后显示折叠按钮，点击可展开

### 5.2 层级切换
- 支持在"项目部"和各工作组之间切换视图
- 切换时数据联动更新：合同概览、任务统计、待办事项

### 5.3 待办事项分类

| Tab | 内容 | 标识颜色 |
|-----|------|----------|
| 待办事项 | 合约任务相关的待办 | 蓝色 |
| 临时任务 | 临时派发的任务 | 紫色 |

### 5.4 今日动态规则
- 按时间倒序展示最新活动
- 支持展开/收起更多历史动态
- 不同类型活动使用不同颜色圆点标识

## 六、创建项目规则

### 6.1 表单验证规则

| 字段 | 验证规则 | 错误提示 |
|------|----------|----------|
| 客户姓名 | 非空 | 请输入客户姓名 |
| 联系方式 | 11位手机号，1开头 | 请输入正确的手机号码 |
| 所在区域 | 省/市/区 + 详细地址非空 | 请选择完整的区域信息 |
| 房屋面积 | 大于0 | 请输入房屋面积 |
| 房屋类别 | 必选 | 请选择房屋类别 |
| 房型 | 必选 | 请选择房型 |

### 6.2 附件上传规则
- 支持图片、PDF、Word等格式
- 上传后显示缩略图
- 支持删除已上传附件

### 6.3 风格选择规则
- 支持多选：中式、法式、现代、轻奢、简约、工业、美式
- 选中/取消选中通过点击切换

---

## 七、项目信息编辑规则

### 7.1 编辑权限规则
- 默认查看模式，所有字段禁用
- 点击"编辑"按钮后，对应区域字段变为可编辑
- 支持分区域独立编辑：客户信息、房屋信息、计划时间、风格喜好、附件
- 保存后自动切换回查看模式

### 7.2 房屋类别选项
- 毛坯房（默认选中）
- 精装房
- 旧房改造

### 7.3 房型选项
- 别墅
- 平层（默认选中）

---

## 八、任务列表页规则

### 8.1 筛选维度

| 筛选维度 | 选项 |
|----------|------|
| 合同 | 全部合同 / 水电 / 泥瓦 / 木工 / 油漆 |
| 任务类型 | 全部类型 / 合约任务 / 临时任务 |
| 状态 | 全部状态 / 待开始 / 确认中 / 已完成 |

### 8.2 搜索规则
- 支持按任务名称模糊搜索
- 实时过滤任务列表
- 搜索结果数量实时更新

### 8.3 统计汇总
- 全部任务数
- 待开始任务数
- 确认中任务数
- 已完成任务数

### 8.4 展开/收起规则
- 支持按合同收起/展开所有任务
- 支持按阶段收起/展开任务列表
- "展开全部"/"收起全部"按钮控制全局展开状态
