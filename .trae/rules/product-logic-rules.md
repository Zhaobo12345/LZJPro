# 家装平台小程序 - 产品逻辑规则

## 生效方式
始终生效

---

## 一、项目层级架构规则

### 1.1 项目层级定义
平台支持最多4层项目分解结构：

| 层级 | 名称 | 说明 | 代码标识 |
|------|------|------|----------|
| Level 1 | 项目部 | 最顶层项目组织 | level-0 |
| Level 2 | 工作组 | 专业分工组（水电/泥瓦/木工/油漆） | level-1 |
| Level 3 | 施工组 | 工作组下的具体施工小组 | level-2 |
| Level 4 | 任务包 | 最小执行单元 | 任务项 |

### 1.2 节点操作权限规则
- 只有项目经理角色才能创建/删除架构节点
- 创建施工组前，必须先确认所属工作组已创建合同
- 删除工作组时，会级联删除其下所有施工组和相关合同

### 1.3 合同与架构绑定规则
- 每个工作组（Level 2）最多关联一个合同
- 创建施工组前，必须先为该工作组创建合同
- 合同状态包括：拟定中、待签署、执行中、已完成

### 1.4 项目树交互规则
- 点击节点可展开/收起子节点
- 支持添加工作组、添加施工组、邀请成员、删除等操作
- 节点展开/收起有动画过渡效果

## 二、合同管理规则

### 2.1 合同状态定义

| 状态 | 显示文字 | 说明 |
|------|----------|------|
| draft | 拟定中 | 合同已创建，请前往电脑端完善 |
| pending_review | 待审核 | PC端特有：合同已提交，等待运营人员审核 |
| review_rejected | 审核驳回 | PC端特有：审核未通过，需修改后重新提交 |
| confirming_sender | 待对方确认 | 已发起确认申请，等待对方确认 |
| confirming_receiver | 待我方确认 | 对方已确认，请我方最终确认 |
| confirmed | 已确认 | 双方已确认，请上传签约文件，上传后合同正式生效 |
| signed | 已签约 | 合同已正式生效，可发起变更 |
| changing | 变更中 | 变更申请已发起，等待对方确认（仅阶段任务变更） |
| change_confirming | 待确认变更 | 对方发起变更，请确认或驳回（仅阶段任务变更） |
| change_platform_reviewing | 变更审核中 | 变更申请已提交，平台运营人员正在审核中（含其他内容变更） |
| change_platform_rejected | 变更审核驳回 | 变更申请审核未通过，请根据驳回原因修改后重新提交 |
| change_confirming_sender | 变更确认中 | 平台审核已通过，等待对方确认变更内容 |
| change_confirming_receiver | 待确认变更 | 对方发起变更申请，平台审核已通过，请确认或驳回 |
| change_signing_wait | 变更签约中 | 变更已确认，请上传线下已签约的合同变更文件，上传后变更正式生效 |

### 2.1.1 PC端合同审核流程（小程序端不涉及）

**审核流程说明：**
- 当合同一方在"拟定中"状态提交合同时，合同进入"待审核"状态
- 运营人员对合同进行审核，审核结果为"通过"或"驳回"
- 审核通过后，合同进入"待对方确认"状态，另一方可以进行确认
- 审核驳回后，合同退回"拟定中"状态，需修改后重新提交

**审核流程状态流转：**
```
拟定中 → [提交审核] → 待审核 → [审核通过] → 待对方确认
                              ↓
                         [审核驳回]
                              ↓
                           拟定中
```

**运营人员审核权限：**
- 仅运营角色可进行合同审核操作
- 审核时可查看合同全部内容
- 驳回时需填写驳回原因

### 2.2 合同状态流转规则

#### 2.2.1 PC端状态流转（含审核流程）

| 当前状态 | 允许操作 | 下一状态 | 操作角色 |
|----------|----------|----------|----------|
| 拟定中 | 提交审核 | 待审核 | 合同创建方 |
| 待审核 | 审核通过 | 待对方确认 | 运营人员 |
| 待审核 | 审核驳回 | 拟定中 | 运营人员 |
| 待对方确认 | 撤回确认 | 拟定中 | 合同创建方 |
| 待我方确认 | 确认合同 | 已确认 | 合同确认方 |
| 待我方确认 | 驳回修改 | 拟定中 | 合同确认方 |
| 已确认 | 上传签约文件 | 已签约 | 任一方 |
| 已签约 | 发起变更 | 变更中 | 任一方 |
| 变更中 | 确认变更 | 已签约 | 另一方 |
| 变更中 | 撤回变更 | 已签约 | 发起方 |

#### 2.2.2 小程序端状态流转（无审核流程）

| 当前状态 | 允许操作 | 下一状态 |
|----------|----------|----------|
| 拟定中 | 提交确认 | 待对方确认 |
| 待对方确认 | 撤回确认 | 拟定中 |
| 待我方确认 | 确认合同 | 已确认 |
| 待我方确认 | 驳回修改 | 拟定中 |
| 已确认 | 上传签约文件 | 已签约 |
| 已签约 | 发起变更（仅阶段任务） | 变更中 |
| 已签约 | 发起变更（含其他内容） | 变更审核中 |
| 变更中 | 确认变更 | 已签约 |
| 变更中 | 撤回变更 | 已签约 |
| 变更审核中 | 撤回变更申请 | 已签约 |
| 变更审核驳回 | 重新发起变更 | 变更审核中 |
| 变更确认中 | 撤回变更 | 已签约 |
| 待确认变更 | 确认变更 | 变更签约中 |
| 待确认变更 | 驳回变更 | 已签约 |
| 变更签约中 | 上传签约文件 | 已签约 |

### 2.3 合同签约双方状态同步规则

| 合同状态 | 甲方状态 | 乙方状态 |
|----------|----------|----------|
| 拟定中 | 编辑中 | 待确认 |
| 待对方确认 | 已确认 | 确认中 |
| 待我方确认 | 已确认 | 确认中 |
| 已确认 | 已确认 | 已确认 |
| 已签约 | 已签约 | 已签约 |

### 2.4 合同变更规则
- 发起变更后，阶段任务暂停流转
- 变更需要双方确认后生效
- 变更生效后生成新的版本记录
- 合同正文修改需在电脑端操作（移动端仅查看）

### 2.4.1 变更类型分类

**变更类型分为两种：**

| 变更类型 | 说明 | 流程 |
|----------|------|------|
| 仅阶段任务变更 | 仅修改阶段任务内容 | 双方确认后生效（无需平台审核） |
| 含其他内容变更 | 包含合同金额、正文或附件变更 | 平台审核 → 双方确认 → 重新签约 |

### 2.4.2 仅阶段任务变更流程

**流程图：**
```
已签约 → [发起变更] → 变更中 → [对方确认] → 已签约
                      ↓
                 [对方驳回]
                      ↓
                   已签约
```

**状态说明：**

| 状态 | 显示文字 | 说明 |
|------|----------|------|
| changing | 变更中 | 变更申请已发起，等待对方确认（阶段任务已暂停流转） |
| change_confirming | 待确认变更 | 对方发起变更申请，请确认或驳回（阶段任务已暂停流转） |

### 2.4.3 含其他内容变更流程

**流程图：**
```
已签约 → [发起变更] → 变更审核中 → [平台审核通过] → 变更确认中 → [双方确认] → 变更签约中 → [上传签约文件] → 已签约
                      ↓                    ↓                    ↓                      
                 [撤回申请]          [平台审核驳回]        [驳回变更]                      
                      ↓                    ↓                    ↓                      
                   已签约              已签约               已签约                    
```

**状态说明：**

| 状态 | 显示文字 | 说明 |
|------|----------|------|
| change_platform_reviewing | 变更审核中 | 变更申请已提交，平台运营人员正在审核中，审核通过后将通知合约方确认 |
| change_platform_rejected | 变更审核驳回 | 变更申请审核未通过，请根据驳回原因修改后重新提交 |
| change_confirming_sender | 变更确认中 | 平台审核已通过，等待对方确认变更内容（阶段任务已暂停流转） |
| change_confirming_receiver | 待确认变更 | 对方发起变更申请，平台审核已通过，请确认或驳回（阶段任务已暂停流转） |
| change_signing_wait | 变更签约中 | 变更已确认，请上传线下已签约的合同变更文件，上传后变更正式生效 |

### 2.4.4 变更流程状态流转规则

#### 仅阶段任务变更状态流转

| 当前状态 | 允许操作 | 下一状态 | 操作角色 |
|----------|----------|----------|----------|
| 已签约 | 发起变更（仅阶段任务） | 变更中 | 任一方 |
| 变更中 | 撤回变更 | 已签约 | 发起方 |
| 待确认变更 | 确认变更 | 已签约 | 另一方 |
| 待确认变更 | 驳回变更 | 已签约 | 另一方 |

#### 含其他内容变更状态流转

| 当前状态 | 允许操作 | 下一状态 | 操作角色 |
|----------|----------|----------|----------|
| 已签约 | 发起变更（含其他内容） | 变更审核中 | 任一方 |
| 变更审核中 | 撤回变更申请 | 已签约 | 发起方 |
| 变更审核中 | 审核通过 | 变更确认中 | 平台运营 |
| 变更审核中 | 审核驳回 | 变更审核驳回 | 平台运营 |
| 变更审核驳回 | 重新发起变更 | 变更审核中 | 发起方 |

### 2.4.5 变更类型选择规则

**选择界面：**
- 发起变更页面顶部显示变更类型选择区域
- 默认选中"仅阶段任务变更"
- 两种类型以单选卡片形式展示

**类型说明：**
- **仅阶段任务变更**：仅修改阶段任务，无需平台审核，双方确认后生效
- **含其他内容变更**：包含合同金额、正文或附件变更，需经平台审核、双方确认、上传签约文件后生效

**选择影响：**
- 选择"仅阶段任务变更"时，提交后进入"变更中"状态
- 选择"含其他内容变更"时，提交后进入"变更审核中"状态

### 2.5 发起变更页面规则

#### 2.5.1 页面头部
- 页面标题右侧显示小程序默认头部信息（"..."与关闭图标）
- 点击"..."可显示更多选项（保存草稿、查看历史变更、导出变更记录）
- 点击关闭图标可关闭变更页面

#### 2.5.2 阶段按序执行配置
- **配置入口**：每个阶段头部都有"按序执行"开关
- **添加新阶段**：新建阶段默认不按序执行，可手动开启
- **编辑阶段**：支持修改阶段名称和按序执行配置
- **阶段操作按钮**：
  - 添加任务按钮：显示"+ 添加任务"（图标+文字）
  - 删除阶段按钮：显示"× 删除阶段"（图标+文字）
  - 按钮样式：圆角胶囊形状，带背景色
- **按序执行修改限制**：
  - 如果阶段下存在已执行任务（已完成/进行中），不支持修改按序执行状态
  - 点击开关提示"当前阶段存在已执行的任务，不支持修改按序执行状态"
  - 开关显示为禁用状态（半透明、不可点击）
- **按序执行规则**：
  - 开启按序执行时，任务按完成状态自动排序
  - 已完成任务排在前面，未完成任务排在后面
  - 保持已完成任务的原有顺序，保持未完成任务的原有顺序

#### 2.5.3 添加新任务弹窗
添加新任务时，弹窗包含以下字段（无需选择合同）：
- 任务名称（必填）
- 执行人（选填，支持搜索选择）
  - 提供搜索输入框，可按姓名或角色搜索
  - 支持模糊搜索，实时过滤选项
  - 可不选择执行人，创建无执行人任务
- 确认人（选填，最多5人，支持多选和搜索）
  - 提供搜索输入框，可按姓名或角色搜索
  - 支持模糊搜索，实时过滤选项
  - 已选确认人以标签形式展示，支持删除
- 执行标准（必填，文本域）
- 确认标准（必填，文本域）
- 担责标准（必填，文本域）

#### 2.5.4 任务查看与编辑
- **查看任务详情**：点击任务名称可查看任务详情（弹窗展示）
  - 显示内容：任务名称、执行人、确认人、执行标准、确认标准、担责标准
  - 只读模式，不可编辑
  - 变更前的任务，各字段显示原有值
  
- **编辑任务**：点击任务右侧的"✎"图标可编辑任务
  - 弹窗展示所有字段，支持修改
  - 执行人和确认人支持搜索选择
  - 保存后更新任务信息

#### 2.5.5 任务删除限制
- **未开始的任务**：支持删除
- **进行中的任务**：不支持删除
  - 删除按钮置灰或隐藏
  - 如点击删除，提示"进行中的任务不支持删除"
  - **示例效果**：任务名称前显示"⏳"图标，任务名称显示为警告色
- **已完成的任务**：不支持删除
  - 删除按钮置灰或隐藏
  - 如点击删除，提示"已完成的任务不支持删除"
  - **示例效果**：任务名称前显示"✓"图标，任务名称显示为成功色

#### 2.5.6 阶段删除限制
- **包含未开始任务的阶段**：支持删除
- **包含进行中任务的阶段**：不支持删除
  - 删除按钮置灰或隐藏
  - 如点击删除，提示"当前阶段存在进行中的任务，不支持删除"
- **包含已完成任务的阶段**：不支持删除
  - 删除按钮置灰或隐藏
  - 如点击删除，提示"当前阶段存在已完成的任务，不支持删除"
- **示例效果**：包含已完成/进行中任务的阶段，删除按钮置灰并显示提示信息

#### 2.5.7 任务编辑限制
- **未开始的任务**：支持编辑所有字段
- **进行中的任务**：支持编辑部分字段（执行人、确认人）
- **已完成的任务**：不支持编辑
  - 编辑按钮置灰或隐藏
  - 如点击编辑，提示"已完成的任务不支持编辑"

#### 2.5.8 人员搜索功能
- **搜索范围**：执行人和确认人选择器
- **搜索方式**：
  - 支持按姓名搜索
  - 支持按角色搜索
  - 支持模糊搜索
- **交互方式**：
  - 输入搜索关键词，实时过滤选项
  - 点击选项完成选择
  - 确认人支持多选，已选人员以标签形式展示

## 三、任务管理规则

### 3.1 任务状态定义

#### 3.1.1 基本状态流程

任务的基本流程为：**待开始 → 执行中 → 确认中 → 已完成**

| 状态 | 说明 | 按钮显示 |
|------|------|----------|
| configuring | 待配置 | 提交任务 |
| pending | 待开始 | 上传执行记录 / 提交给确认人 |
| in_progress | 执行中 | 上传执行记录 / 提交给确认人 |
| confirming | 确认中 | 驳回 / 确认通过 |
| completed | 已完成 | 查看记录 / 查看评价 |

#### 3.1.2 细分状态（含驳回流程）

| 状态 | 说明 | 按钮显示 |
|------|------|----------|
| confirming | 确认中（首次） | 驳回 / 确认通过 |
| confirming_after_reject | 确认中（被驳回后） | 驳回 / 确认通过 |
| completed | 已完成（无驳回） | 查看记录 / 查看评价 |
| completed_with_reject | 已完成（含驳回） | 查看记录 / 查看评价 |

#### 3.1.3 前置任务阻塞状态

| 状态 | 说明 | 按钮显示 |
|------|------|----------|
| pending_blocked | 待开始（前置阻塞） | 前置任务未完成（禁用状态） |

**前置任务阻塞规则：**

1. **触发条件**：
   - 阶段设置了"按序执行"
   - 当前任务存在未完成的前置任务

2. **阻塞效果**：
   - 显示前置任务阻塞提示卡片
   - 列出所有未完成的前置任务及其状态
   - 执行按钮置灰禁用，显示"🔒 前置任务未完成"
   - 流程进度显示"前置任务阻塞中"

3. **前置任务状态显示**：
   | 状态 | 颜色 | 说明 |
   |------|------|------|
   | 待开始 | 橙色 | 前置任务尚未开始 |
   | 进行中 | 蓝色 | 前置任务正在执行 |
   | 已完成 | 绿色 | 前置任务已完成（不显示在阻塞列表） |

4. **解除阻塞**：
   - 所有前置任务完成后自动解除阻塞
   - 系统自动通知执行人可以开始执行
   - 任务状态自动变为"待开始"

5. **提示信息**：
   - 阻塞卡片显示："当前任务设置了按序执行，需等待前置任务完成后才能开始执行"
   - 底部提示："💡 前置任务完成后将自动通知您"

### 3.2 任务执行流程
- 任务创建后，状态为"待配置"
- 配置执行人后，可提交任务，状态变为"待开始"
- 执行人点击"上传执行记录"上传内容后，状态变为"执行中"
- 执行人点击"提交给确认人"后，状态变为"确认中"
- 所有确认人确认后，状态变为"已完成"
- 任一确认人驳回，任务退回执行人重新执行，状态变为"待开始"
- 执行人重新提交执行记录后，状态变为"确认中（被驳回后）"
- 所有确认人确认后，状态变为"已完成（含驳回）"

### 3.2.1 待配置状态规则
**待配置状态（configuring）**：
- 任务创建后的初始状态
- 状态提示：显示"请配置执行人（必填）和确认人（选填）"
- 执行人：必填，支持添加/修改
- 确认人：选填，最多5人，支持添加/删除
- 确认人列表展示：
  - 无确认人时：显示"暂无确认人，点击添加"提示
  - 有确认人时：只显示姓名和角色，不显示状态
  - 删除按钮默认显示，点击可移除确认人
- 底部按钮：
  - 无执行人时：显示"请先选择执行人"（灰色禁用状态）
  - 有执行人时：显示"提交任务"按钮
- 提交任务流程：
  - 点击"提交任务"显示确认弹窗
  - 弹窗显示执行人和确认人信息
  - 提示"提交后执行人和确认人将不可修改"
  - 确认后发送通知，状态变为"待执行"
  - 执行人和确认人锁定，不可修改

### 3.2.2 待开始状态规则
**待开始状态（pending）**：
- 执行人视角底部显示两个按钮：
  - "上传执行记录"按钮（次要按钮）
  - "提交给确认人"按钮（主要按钮）
- 非执行人视角显示"等待执行人执行"提示

### 3.2.3 执行中状态规则
**执行中状态（in_progress）**：
- 执行人已上传执行记录，但尚未提交给确认人
- 执行记录卡片显示已上传的内容
- 执行人可继续上传执行记录（支持编辑/删除）
- 执行人视角底部显示两个按钮：
  - "上传执行记录"按钮（次要按钮）
  - "提交给确认人"按钮（主要按钮）
- 非执行人视角显示"执行人正在执行中..."提示
- 流程进度显示为"执行中"

### 3.2.4 上传执行记录功能
**上传执行记录弹窗**：
- 支持添加图片、视频或音频
- 执行说明为必填项
- 点击"确认上传"后：
  - 执行记录展示在执行记录卡片内
  - 任务状态变为"执行中"
  - 执行人可继续编辑或删除已上传的记录

### 3.2.5 提交给确认人功能
**提交确认弹窗**：
- 显示确认人列表
- 提示"提交后不支持修改执行记录"
- 点击"确认提交"后：
  - 任务状态变为"确认中"
  - 通知确认人进行确认
  - 执行记录锁定，不可修改

### 3.2.6 确认中状态区分
**首次确认中（confirming）**：
- 执行记录只显示当前提交的内容
- 不显示任何驳回信息
- 确认人只需确认当前执行内容
- 显示确认进度条

**被驳回后确认中（confirming_after_reject）**：
- 执行记录按时间倒序展示所有历史记录
- 显示上次被驳回的执行记录及驳回原因
- 显示最新的执行记录（当前有效）
- 确认人可参考历史记录进行确认
- 确认记录显示完整的确认历史（包括驳回记录）
- 显示确认进度条

### 3.2.3 已完成状态区分
**已完成（无驳回）**：
- 执行记录只显示最终通过的执行内容
- 确认记录只显示确认人的确认记录
- 不显示任何驳回相关信息
- 不显示确认进度条
- 提供"查看历史记录"按钮（但无历史记录）

**已完成（含驳回）**：
- 执行记录按时间倒序展示所有历史记录
- 显示被驳回的执行记录及驳回原因
- 显示最终通过的执行记录（当前有效）
- 确认记录显示完整的确认历史（包括驳回操作记录）
- 不显示确认进度条
- 提供"查看历史记录"按钮，可查看所有历史记录

### 3.2.4 确认进度条显示规则
- **显示确认进度条的状态**：
  - 确认中（首次）
  - 确认中（被驳回后）
- **不显示确认进度条的状态**：
  - 待配置
  - 待执行
  - 驳回后待开始
  - 已完成（无驳回）
  - 已完成（含驳回）

### 3.3 确认人机制
- 每个任务可设置最多5个确认人
- 确认人可给出评分（1-5星）
- 确认进度显示格式：`确认进度：已确认数/总确认人数`
- 确认时可上传图片、视频、音频

### 3.4 三层标准规则
每个任务包含三个维度的标准：

| 标准类型 | 用途 | 展示位置 |
|----------|------|----------|
| 执行标准 | 指导执行人如何完成任务 | 可折叠卡片 |
| 确认标准 | 指导确认人如何验收 | 可折叠卡片 |
| 担责标准 | 明确责任归属 | 可折叠卡片 |

### 3.5 无执行人任务规则
- **创建规则**：任务执行人为选填项，可不选择执行人
- **执行人限制**：每个任务只能有一个执行人
- **待配置状态展示**：
  - 显示"暂无执行人，点击添加"提示
  - 点击后弹出人员选择器，可选择项目内成员
  - 支持按姓名搜索过滤
  - **不显示"提交任务"按钮**（显示"请先选择执行人"禁用状态）
- **添加执行人功能**：
  - 点击"暂无执行人，点击添加"弹出人员选择器
  - 选择器显示项目内所有成员
  - 支持按姓名或角色搜索
  - 选择后自动更新执行人信息
  - 选择后显示"提交任务"按钮
- **修改执行人功能**：
  - 已有执行人时，按钮显示为"修改执行人"（图标✎）
  - 点击后弹出人员选择器，可更换执行人
  - 选择后更新执行人信息
- **添加确认人功能**：
  - 点击"添加确认人"弹出人员选择器
  - 选择器显示项目内所有成员
  - 支持按姓名或角色搜索
  - 支持多选，最多选择5人
  - 已选人员显示勾选状态
  - 确认后更新确认人列表
- **删除确认人功能**：
  - 确认人列表项hover时显示删除按钮（红色圆形×）
  - 点击删除按钮移除确认人
  - 至少保留一位确认人
  - 删除后更新确认人数量显示
- **提交任务功能**：
  - 选择执行人后，显示"提交任务"按钮
  - 点击显示确认弹窗，展示执行人和确认人信息
  - 提示"提交后执行人和确认人将不可修改"
  - 确认后发送通知，状态变为"待执行"
  - 执行人和确认人锁定，不可修改
- **待执行状态**：
  - 执行人视角：显示"开始执行"按钮
  - 非执行人视角：显示"等待执行人执行"提示
  - 执行人/确认人不可修改
- **驳回后待开始状态**：
  - 驳回后待开始状态必须有执行人（因为任务已被执行过）
  - 执行人视角：显示"重新执行"按钮
  - 非执行人视角：显示"等待执行人重新执行"提示

### 3.6 执行记录保留规则

#### 3.6.1 记录状态定义

| 状态 | 说明 | 显示样式 |
|------|------|----------|
| 有效 | 当前有效的执行记录 | 正常显示 |
| 已驳回 | 被确认人驳回的记录 | 灰色显示，带"已驳回"标签 |
| 已废弃 | 因其他原因废弃的记录 | 灰色显示，带"已废弃"标签 |

#### 3.6.2 执行轮次管理
- 每次执行都有独立的轮次编号（从1开始递增）
- 任务首次执行：轮次为1
- 被驳回后重新执行：轮次为2，以此类推
- 每个轮次可包含多条记录（开始执行、进度更新、完成执行等）

#### 3.6.3 驳回后重新执行规则
- 任务被驳回时，将该轮次的所有执行记录标记为"已驳回"
- 执行人重新执行时，创建新的执行轮次
- 历史执行记录完整保留，按时间倒序展示
- 展示时需区分：
  - 当前有效记录：正常显示，突出展示
  - 历史驳回记录：灰色显示，可折叠查看

#### 3.6.4 执行记录展示规则
- 默认展示当前有效轮次的执行记录
- 提供"查看历史记录"按钮，可展开查看所有轮次
- 每个轮次独立展示，包含：
  - 轮次编号
  - 提交时间
  - 执行内容
  - 附件（图片、视频、音频）
  - 状态标签（有效/已驳回）
  - 驳回原因（如有）

#### 3.6.5 不同状态下的展示规则

**待开始状态**：
- 如有历史驳回记录，显示最后一次被驳回的执行记录
- 显示驳回原因、驳回人、驳回时间
- 不显示"查看历史记录"按钮

**首次确认中状态**：
- 只显示当前执行记录（第1轮次）
- 不显示任何驳回信息
- 不显示"查看历史记录"按钮

**被驳回后确认中状态**：
- 按时间倒序展示所有执行记录
- 第2轮次（最新）：显示"当前有效"标签，正常显示
- 第1轮次（已驳回）：显示"已驳回"标签，灰色显示，附带驳回原因
- 确认记录显示完整的确认历史（包括驳回操作记录）
- 不显示"查看历史记录"按钮

**已完成（无驳回）状态**：
- 只显示最终通过的执行记录（第1轮次）
- 确认记录只显示确认人的确认记录
- 不显示任何驳回相关信息
- 不显示"查看历史记录"按钮

**已完成（含驳回）状态**：
- 按时间倒序展示所有执行记录
- 第2轮次（最新）：显示"当前有效"标签，正常显示
- 第1轮次（已驳回）：显示"已驳回"标签，灰色显示，附带驳回原因
- 确认记录显示完整的确认历史（包括驳回操作记录）
- 不显示"查看历史记录"按钮

#### 3.6.6 数据库字段说明
- `execution_round`：执行轮次，INT类型，从1开始
- `record_status`：记录状态，TINYINT类型（1-有效 2-已驳回 3-已废弃）
- `reject_reason`：驳回原因，VARCHAR(500)
- `reject_time`：驳回时间，DATETIME
- `reject_user_id`：驳回人ID，BIGINT

### 3.7 记录收起展开规则

#### 3.7.1 执行记录收起展开
- **默认状态**：展开显示执行记录内容
- **收起功能**：
  - 点击执行记录卡片标题可收起内容
  - 收起后显示向下箭头图标
  - 收起后内容隐藏，只显示标题
- **展开功能**：
  - 点击已收起的卡片标题可展开内容
  - 展开后显示向上箭头图标
  - 展开后显示完整执行记录内容

#### 3.7.2 确认记录收起展开
- **默认状态**：展开显示确认记录内容
- **收起功能**：
  - 点击确认记录卡片标题可收起内容
  - 收起后显示向下箭头图标
  - 收起后内容隐藏，只显示标题
- **展开功能**：
  - 点击已收起的卡片标题可展开内容
  - 展开后显示向上箭头图标
  - 展开后显示完整确认记录内容

#### 3.7.3 交互效果
- 收起/展开有平滑过渡动画
- 箭头图标旋转动画
- 内容高度渐变动画

## 四、临时任务规则

### 4.1 临时任务生命周期
待接收 → 待开始 → 确认中 → 已完成
↓
已拒绝

### 4.2 创建临时任务规则
- **表单验证规则**：

| 字段 | 验证规则 | 错误提示 |
|------|----------|----------|
| 任务名称 | 必填 | 请输入任务名称 |
| 关联合同 | 必填 | 请选择关联合同 |

| 确认人 | 选填，最多5人 | - |
| 执行标准 | 必填 | 请输入执行标准 |
| 确认标准 | 必填 | 请输入确认标准 |
| 担责标准 | 必填 | 请输入担责标准 |

- **执行人非必填说明**：
  - 可创建无执行人的临时任务

## 五、项目详情页规则

### 5.1 快捷入口
- 四个快捷入口：项目信息、架构、成员、资料
- 支持收起/展开快捷入口（滚动时自动收起）
- 收起后显示折叠按钮，点击可展开

### 5.2 层级切换
- 支持在"项目部"和各工作组之间切换视图
- 切换时数据联动更新：合同概览、任务统计、待办事项

### 5.3 待办事项分类

| Tab | 内容 | 标识颜色 |
|-----|------|----------|
| 待办事项 | 合约任务相关的待办 | 蓝色 |
| 临时任务 | 临时派发的任务 | 紫色 |

### 5.4 今日动态规则
- 按时间倒序展示最新活动
- 支持展开/收起更多历史动态
- 不同类型活动使用不同颜色圆点标识

### 5.5 待办事项和临时任务展示优化
- **查看全部待办**：
  - 使用渐变背景色（浅蓝色渐变）
  - 添加边框和阴影效果
  - hover时有上浮动画
  - 字体加粗，箭头图标加大
  
- **创建临时任务**：
  - 使用渐变背景色（浅紫色渐变）
  - 添加边框和阴影效果
  - hover时有上浮动画
  - 图标使用渐变色，字体加粗
  
- **查看全部临时任务**：
  - 与"查看全部待办"样式一致
  - 使用浅蓝色渐变背景

### 5.6 项目资料页面规则

#### 5.6.1 页面入口
- 项目详情页金刚位"资料"点击进入
- 显示当前项目下所有文件资料

#### 5.6.2 文件分类展示
- **全部**：显示所有类型文件
- **图片**：仅显示图片文件（jpg、png、gif等）
- **视频**：仅显示视频文件（mp4、avi等）
- **音频**：仅显示音频文件（mp3、wav等）
- **文档**：仅显示文档文件（pdf、doc、xls等）

#### 5.6.3 文件来源分类
- **项目信息附件**：创建项目时上传的附件
- **合同附件**：按合同分组展示
  - 水电合同
  - 泥瓦合同
  - 木工合同
  - 油漆合同
- **阶段任务附件**：按阶段和任务分组展示

#### 5.6.4 权限控制
- **有权限的合同**：正常显示文件内容
- **无权限的合同**：
  - 合同区域显示灰色
  - 文件显示为加密状态
  - 点击提示"无权限查看此合同资料"
  - 显示"无权限"标签
- **权限提示**：
  - 页面右下角显示浮动提示图标（⚠️）
  - 点击图标弹出权限说明气泡
  - 点击其他区域自动关闭气泡

#### 5.6.5 文件展示样式
- **图片文件**：显示缩略图
- **视频文件**：显示封面图+播放图标
- **音频文件**：显示渐变色音频图标
- **文档文件**：显示文档类型图标（PDF/DOC/XLS等）

#### 5.6.6 文件信息展示
- 文件名称（超出显示省略号）
- 文件来源（项目信息/合同名称/阶段任务）
- 上传时间或文件大小

#### 5.6.7 搜索功能
- **搜索入口**：页面顶部固定搜索栏
- **搜索方式**：
  - 支持按文件名称模糊搜索
  - 实时过滤文件列表（输入即搜索）
  - 显示搜索结果数量
- **搜索结果**：
  - 匹配的文件正常显示
  - 不匹配的文件隐藏
  - 无结果时显示"未找到相关文件"提示
- **搜索交互**：
  - 清空搜索框恢复全部文件显示
  - 切换分类Tab时自动清空搜索

#### 5.6.8 文件预览
- **图片**：点击全屏预览，支持缩放
- **视频**：点击播放视频
- **音频**：点击播放音频
- **文档**：点击预览文档内容

## 六、创建项目规则

### 6.1 表单验证规则

| 字段 | 验证规则 | 错误提示 |
|------|----------|----------|
| 客户姓名 | 非空 | 请输入客户姓名 |
| 联系方式 | 11位手机号，1开头 | 请输入正确的手机号码 |

### 10.8.1 PC端合同审核功能规则

**审核入口**：
- 合同列表页面 → 待审核状态合同 → 审核按钮
- 合同详情页面 → 待审核状态 → 审核操作区

**审核人员权限**：
- 仅运营角色用户可进行合同审核
- 其他角色用户无法看到审核操作按钮

**审核操作**：
| 操作 | 说明 | 结果 |
|------|------|------|
| 审核通过 | 确认合同内容无误 | 合同状态变为"待对方确认" |
| 审核驳回 | 合同内容存在问题 | 合同状态退回"拟定中"，需填写驳回原因 |

**审核弹窗内容**：
- 合同基本信息预览
- 合同正文预览
- 阶段任务预览
- 审核操作按钮：通过、驳回
- 驳回时需填写驳回原因（必填）

**审核记录**：
- 记录审核人、审核时间、审核结果
- 驳回时记录驳回原因
- 在合同详情页"操作记录"中展示

### 10.9 跨端访问规则

#### 10.9.1 小程序端打开PC端

**入口位置**：合同详情页面右上角"..."菜单

**操作流程**：
1. 点击"..."菜单，选择"在电脑端打开"
2. 弹窗显示二维码和链接
3. 用户可选择：
   - 扫描二维码：使用微信扫一扫，在电脑端浏览器打开
   - 复制链接：手动复制链接到电脑浏览器打开

**弹窗内容**：
- 二维码：包含合同详情页面的PC端链接
- 链接地址：格式为 `https://lzjpro.com/contract/{合同编号}`
- 复制按钮：点击复制链接到剪贴板

#### 10.9.2 PC端扫码登录

**登录页面**：`pc-login.html`

**登录方式**：
1. 扫码登录（推荐）
2. 账号密码登录
3. 短信验证登录

**扫码登录流程**：
1. 打开PC端登录页面，显示二维码
2. 使用微信扫一扫扫描二维码
3. 手机端显示登录确认页面
4. 确认登录后，PC端自动跳转到目标页面

**二维码有效期**：
- 默认有效期：120秒
- 过期后显示刷新按钮，点击可刷新二维码

**登录成功后**：
- 显示用户头像和姓名
- 3秒后自动跳转到目标页面
- 若从合同链接进入，跳转到对应合同详情页
