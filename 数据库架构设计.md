# 全透明家装平台 - 数据库架构设计

## 文档说明
本文档详细描述了家装平台的数据库架构设计，包括数据库选型、表结构设计、索引设计、数据字典等内容。

---

## 一、数据库选型与架构

### 1.1 数据库选型

#### 1.1.1 关系型数据库：MySQL 8.0
**选型理由**：
- 成熟稳定，社区活跃
- 支持事务，数据一致性有保障
- 支持复杂查询和关联
- 运维成本低

**应用场景**：
- 核心业务数据存储
- 事务处理
- 复杂查询

#### 1.1.2 缓存数据库：Redis 7.x
**选型理由**：
- 高性能读写
- 支持多种数据结构
- 支持持久化
- 支持分布式

**应用场景**：
- 会话缓存
- 热点数据缓存
- 分布式锁
- 消息队列

#### 1.1.3 文档数据库：MongoDB
**选型理由**：
- 灵活的数据模型
- 高性能读写
- 支持海量数据
- 支持地理位置索引

**应用场景**：
- 日志数据存储
- 动态数据存储
- 非结构化数据存储

#### 1.1.4 搜索引擎：ElasticSearch 8.x
**选型理由**：
- 强大的全文检索能力
- 支持复杂查询
- 支持聚合分析
- 高性能

**应用场景**：
- 全文检索
- 日志分析
- 数据分析

---

### 1.2 数据库架构设计

#### 1.2.1 分库分表策略

**垂直分库**：
- 用户库（user_db）：用户相关数据
- 项目库（project_db）：项目相关数据
- 合同库（contract_db）：合同相关数据
- 资金库（fund_db）：资金相关数据
- 消息库（message_db）：消息相关数据

**水平分表**：
- 项目表：按年度分表（project_2024, project_2025）
- 任务表：按项目ID分表
- 消息表：按用户ID分表
- 日志表：按月份分表

#### 1.2.2 读写分离

**主从架构**：
- 主库：负责写操作
- 从库：负责读操作
- 主从同步：异步复制

**读写分离策略**：
- 写操作：路由到主库
- 读操作：路由到从库
- 事务操作：路由到主库

## 三、索引设计原则

### 3.1 索引设计原则

1. **主键索引**：所有表都有自增主键
2. **唯一索引**：编码、手机号等唯一字段
3. **普通索引**：常用查询条件字段
4. **组合索引**：多字段组合查询
5. **外键索引**：关联字段建立索引

### 3.2 索引优化建议

1. **避免过度索引**：索引不是越多越好
2. **定期维护索引**：定期重建索引
3. **监控索引使用**：删除未使用的索引
4. **考虑查询频率**：高频查询字段优先索引

## 五、数据库运维建议

### 5.1 备份策略

1. **全量备份**：每天凌晨2点执行
2. **增量备份**：每小时执行一次
3. **日志备份**：每15分钟执行一次
4. **异地备份**：每周同步到异地机房

### 5.2 监控指标

1. **性能指标**：QPS、TPS、响应时间
2. **资源指标**：CPU、内存、磁盘、网络
3. **连接指标**：连接数、活跃连接
4. **慢查询**：慢查询数量、执行时间

### 5.3 优化建议

1. **定期优化表**：每月执行一次OPTIMIZE TABLE
2. **定期分析表**：每周执行一次ANALYZE TABLE
3. **监控慢查询**：优化慢查询SQL
4. **清理历史数据**：定期归档历史数据
