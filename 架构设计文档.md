# 全透明家装共创共治共享平台 - 架构设计文档

## 一、平台定位与核心理念

### 1.1 平台定位
全透明家装共创共治共享平台体系，任务合同化执行

### 1.2 核心理念
- **全透明**：项目数据、资金流向、施工进度、验收结果全程透明
- **共创**：多方参与项目创建、方案设计、任务分解
- **共治**：多方协同治理项目，明确权责边界
- **共享**：资源共享、信息共享、经验共享
- **任务合同化**：每个任务都有明确的合同约束、执行标准、验收标准、担责标准

### 1.3 业务模式
**混合模式**：业主可选择与平台或服务商直接签约，灵活组合

### 1.4 资金管理
**第三方托管**：有信任背书的第三方资金托管，按节点验收后支付

## 二、角色体系设计

### 2.1 核心角色定义

| 角色 | 终端 | 核心职责 | 权限范围 |
|------|------|----------|----------|
| **业主** | C端 | 项目发起、资金提供、验收确认、项目监督 | 项目全流程查看、验收确认、资金支付授权 |
| **项目总** | B端 | 项目总负责、统筹协调、质量把控、进度管理 | 项目全局管理、人员调配、合同审核、进度控制 |
| **设计师** | B端 | 设计方案、图纸输出、材料选型、设计变更 | 设计方案管理、图纸上传、材料推荐 |
| **工长** | B端 | 现场管理、人员安排、材料验收、施工协调 | 施工现场管理、任务分配、进度汇报 |
| **监理** | B端 | 质量监督、节点验收、问题整改、过程记录 | 质量检查、验收确认、问题上报 |
| **施工班组** | B端 | 具体施工、任务执行、进度汇报、问题反馈 | 任务执行、进度汇报、问题上报 |
| **材料商** | B端 | 材料供应、配送安装、售后保障 | 材料订单管理、配送记录、售后处理 |
| **平台管理员** | 管理后台 | 平台运营、用户管理、数据监控、纠纷处理 | 平台全局管理 |

### 3.2 技术栈选型

#### 3.2.1 前端技术栈
- **管理后台PC端**：Vue 3 + Element Plus + TypeScript
- **B端小程序**：微信小程序原生 + uni-app（跨端）
- **C端小程序**：微信小程序原生 + uni-app（跨端）
- **移动端APP**：uni-app（跨平台）或 React Native

#### 3.2.2 后端技术栈
- **核心框架**：Spring Boot 3.x / Spring Cloud Alibaba
- **API网关**：Spring Cloud Gateway
- **服务注册与发现**：Nacos
- **配置中心**：Nacos
- **分布式事务**：Seata
- **消息队列**：RocketMQ / RabbitMQ
- **数据库**：MySQL 8.0（主库）、MongoDB（日志/动态）
- **缓存**：Redis 7.x
- **搜索引擎**：ElasticSearch 8.x
- **对象存储**：阿里云OSS / 腾讯云COS
- **任务调度**：XXL-Job

#### 3.2.3 运维技术栈
- **容器编排**：Kubernetes + Docker
- **监控告警**：Prometheus + Grafana
- **日志中心**：ELK Stack（Elasticsearch + Logstash + Kibana）
- **CI/CD**：Jenkins + GitLab

```sql
    city VARCHAR(50) COMMENT '市',
    district VARCHAR(50) COMMENT '区',
    address VARCHAR(200) COMMENT '详细地址',
    longitude DECIMAL(10,7) COMMENT '经度',
    latitude DECIMAL(10,7) COMMENT '纬度',
    
    -- 房屋信息
    house_type VARCHAR(50) COMMENT '户型：如三室两厅',
    house_area DECIMAL(10,2) COMMENT '房屋面积（平方米）',
    floor_info VARCHAR(50) COMMENT '楼层信息',
    decoration_type TINYINT COMMENT '装修类型：1-毛坯 2-精装 3-旧房改造',
    
    -- 预算信息

    design_budget DECIMAL(12,2) COMMENT '设计预算',
    construction_budget DECIMAL(12,2) COMMENT '施工预算',
    material_budget DECIMAL(12,2) COMMENT '材料预算',
    
    -- 时间信息
    plan_start_date DATE COMMENT '计划开工日期',
    plan_end_date DATE COMMENT '计划完工日期',
    actual_start_date DATE COMMENT '实际开工日期',
    actual_end_date DATE COMMENT '实际完工日期',
    
    -- 项目团队
    owner_id BIGINT COMMENT '业主ID',
    project_manager_id BIGINT COMMENT '项目总ID',
    designer_id BIGINT COMMENT '设计师ID',
    supervisor_id BIGINT COMMENT '监理ID',
    foreman_id BIGINT COMMENT '工长ID',
    
    -- 签约信息
    contract_mode TINYINT COMMENT '签约模式：1-与平台签约 2-与服务商签约',
    contract_party_id BIGINT COMMENT '签约方ID',
    
    -- 其他信息
    project_intro TEXT COMMENT '项目简介',
    special_requirements TEXT COMMENT '特殊要求',
    cover_image VARCHAR(500) COMMENT '项目封面图',
    
    -- 系统字段
    create_by BIGINT COMMENT '创建人',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_by BIGINT COMMENT '更新人',
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    is_deleted TINYINT DEFAULT 0 COMMENT '是否删除：0-否 1-是',
    
    INDEX idx_owner_id (owner_id),
    INDEX idx_project_status (project_status),
    INDEX idx_create_time (create_time)
) COMMENT='项目主表';
```

5. 变更执行（更新计划、合同、任务）

3. 风险应对（制定应对措施）
4. 风险监控（项目过程中持续监控）
5. 风险处置（风险发生时的处置流程）

## 九、总结

本架构设计文档基于家装行业特点，结合用户需求，设计了完整的全透明家装共创共治共享平台架构。核心亮点包括：

1. **全流程透明**：从项目创建到验收结算，全流程数据透明可追溯
2. **任务合同化**：每个任务都有明确的合同约束、执行标准、验收标准、担责标准
3. **多方协同**：支持业主、项目总、设计师、监理、工长、施工班组、材料商等多角色协同
4. **资金安全**：第三方托管机制，保障资金安全
5. **质量保障**：完善的质量巡检、验收、评价体系
6. **售后服务**：完整的质保和售后服务体系

本架构设计可根据实际业务需求和技术条件，分期实施，逐步完善。
