# 全透明家装平台 - 业务模块补充设计

## 文档说明
本文档是对核心业务模块的补充完善，针对家装行业特点和平台定位，新增了必要的业务模块，并对原有模块进行了优化设计。

---

## 一、新增核心业务模块

### 1.1 资金托管模块

#### 1.1.1 模块定位
资金托管模块是平台的核心信任机制，通过第三方托管保障资金安全，按节点验收后支付，解决家装行业资金信任问题。

#### 1.1.2 功能架构

```
资金托管模块
├── 托管账户管理
│   ├── 账户开设
│   ├── 账户信息维护
│   ├── 账户状态管理
│   └── 账户注销
├── 资金充值
│   ├── 充值申请
│   ├── 支付渠道对接
│   ├── 充值确认
│   └── 充值记录查询
├── 资金冻结
│   ├── 冻结申请
│   ├── 冻结审核
│   ├── 冻结执行
│   └── 冻结查询
├── 资金解冻
│   ├── 解冻申请
│   ├── 解冻审核
│   ├── 解冻执行
│   └── 解冻查询
├── 资金划转
│   ├── 划转申请
│   ├── 划转审核
│   ├── 划转执行
│   └── 划转记录查询
├── 资金流水
│   ├── 流水记录
│   ├── 流水查询
│   ├── 流水导出
│   └── 流水统计
└── 对账管理
    ├── 日常对账
    ├── 月度对账
    ├── 差异处理
    └── 对账报告
```

#### 1.1.3 核心业务规则

**资金托管规则**：
1. 项目启动前，业主需将项目总预算的30%充值到托管账户
2. 每个付款节点，业主需提前充值对应金额到托管账户
3. 验收通过后，系统自动触发资金划转
4. 资金冻结期间，业主可查看但不可使用
5. 资金解冻需经过平台审核

**付款节点设置**：
- 首付款：合同签订后支付20%
- 进度款：水电验收后支付30%
- 进度款：泥瓦验收后支付20%
- 进度款：木工验收后支付15%
- 尾款：竣工验收后支付15%

**资金安全保障**：
1. 银行级加密存储
2. 多重身份验证
3. 资金变动实时通知
4. 异常交易自动拦截
5. 资金保险保障

#### 1.1.4 业务流程

**充值流程**：
```
业主 → 充值申请 → 选择支付渠道 → 支付 → 系统确认 → 到账通知
```

**付款流程**：
```
验收通过 → 系统触发付款 → 资金冻结 → 业主确认 → 资金划转 → 到账通知
```

**退款流程**：
```
业主申请 → 平台审核 → 资金解冻 → 退款执行 → 到账通知
```

#### 1.2.3 核心业务规则

**材料选型规则**：
1. 设计师根据设计方案推荐材料
2. 业主可查看材料详情、价格、供应商信息
3. 支持材料对比功能（价格、质量、品牌）
4. 材料确认后自动生成材料清单

**材料采购规则**：
1. 根据施工进度提前采购
2. 采购前需确认库存和价格
3. 采购订单需项目总审核
4. 采购金额超过预算需业主确认

**材料验收规则**：
1. 材料到货后通知工长验收
2. 验收内容包括：数量、质量、规格
3. 验收不合格可拒收或退货
4. 验收合格后拍照记录

**材料结算规则**：
1. 材料验收合格后可申请结算
2. 结算需提供发票和验收单
3. 结算金额需与采购订单一致
4. 结算后更新项目成本

**辅材**：
- 水泥、沙子、电线、水管、板材、钉子、胶水等

**定制材料**：
- 定制家具、定制门窗、定制柜体等

**特殊材料**：


#### 1.3.3 核心业务规则

**巡检计划规则**：
1. 每个项目必须制定巡检计划
2. 日常巡检：每周至少2次
3. 节点巡检：每个施工节点完成后
4. 专项巡检：针对特定质量问题

**巡检内容标准**：

**水电工程巡检**：
- 水管走向是否合理
- 电线铺设是否规范
- 接头处理是否牢固
- 水压测试是否合格
- 电路测试是否正常

**泥瓦工程巡检**：
- 墙面平整度
- 瓷砖铺贴质量
- 防水处理是否到位
- 阴阳角处理

**木工工程巡检**：
- 木制品尺寸精度
- 接缝处理
- 表面光滑度
- 结构牢固性

**油漆工程巡检**：
- 墙面平整度
- 颜色均匀度
- 无流挂、起泡
- 边角处理

**问题整改规则**：
1. 问题发现后立即记录并拍照
2. 问题级别：一般、严重、紧急
3. 整改期限：一般7天、严重3天、紧急24小时
4. 整改完成后需复查验收
5. 整改不合格需重新整改

#### 1.3.4 质量评分体系

**评分维度**：
- 施工工艺（30分）
- 材料质量（20分）
- 现场管理（20分）
- 安全文明（15分）
- 进度控制（15分）

**评分等级**：
- 优秀：90-100分
- 良好：80-89分
- 合格：70-79分
- 不合格：70分以下

**评分应用**：
- 评分结果影响服务商信用等级
- 优秀服务商可获得更多项目机会
- 不合格服务商需整改或下架

3. 售后评价：售后服务完成后

**评价维度**：

**业主评价服务商**：
- 施工质量（30%）
- 服务态度（25%）
- 进度控制（20%）
- 沟通配合（15%）
- 性价比（10%）

**业主评价设计师**：
- 设计水平（35%）
- 服务态度（25%）
- 沟通效率（20%）
- 创新能力（20%）

**业主评价监理**：
- 专业水平（30%）
- 工作态度（30%）
- 问题处理（25%）
- 沟通反馈（15%）

**项目总评价施工班组**：
- 施工质量（40%）
- 进度执行（30%）
- 安全文明（20%）
- 配合度（10%）

**信用等级规则**：
- A级：信用分90-100，优先推荐
- B级：信用分80-89，正常推荐
- C级：信用分70-79，限制接单
- D级：信用分70以下，暂停服务

**信用分计算**：
```
信用分 = 基础分（100分） + 加分项 - 减分项

加分项：
- 优秀评价：+2分/次
- 按时完工：+3分/次
- 零投诉：+5分/月
- 业主推荐：+5分/次

减分项：
- 差评：-3分/次
- 投诉：-5分/次
- 违约：-10分/次
- 安全事故：-20分/次
```

#### 1.4.4 评价展示规则

**公开评价**：
- 评价内容公开显示
- 评价人可选择匿名
- 服务商可回复评价
- 平台可审核评价

**评价排序**：
- 默认按时间倒序
- 可按评分排序
- 可按有用数排序

**评价筛选**：
- 可筛选评分等级

- 可筛选时间范围

## 二、原有模块优化设计

### 2.1 项目创建模块优化

#### 2.1.1 新增功能：智能报价

**功能说明**：
根据房屋面积、装修档次、地区差异，自动估算项目预算。

**报价模型**：
```
总预算 = 基础装修费 + 主材费 + 家具家电费 + 其他费用

基础装修费 = 面积 × 单价 × 地区系数 × 档次系数
主材费 = 基础装修费 × 主材比例
家具家电费 = 面积 × 家具家电单价
其他费用 = 设计费 + 管理费 + 税金
```

**装修档次**：
- 简装：600-800元/㎡
- 精装：1000-1500元/㎡
- 豪装：2000元/㎡以上

**地区系数**：
- 一线城市：1.2-1.5
- 二线城市：1.0-1.2
- 三四线城市：0.8-1.0

#### 2.1.2 新增功能：项目模板

**模板类型**：
- 简装模板：适合出租房、过渡房
- 精装模板：适合自住房
- 豪装模板：适合高端住宅
- 局部改造模板：适合旧房改造

**模板内容**：
- 项目阶段划分
- 任务分解模板
- 材料清单模板
- 预算分配模板

#### 2.1.3 新增功能：项目可行性评估

**评估维度**：
- 预算合理性：预算是否充足
- 工期可行性：工期是否合理

## 三、业务规则补充

### 3.1 项目管理规则

#### 3.1.1 项目状态流转规则

```
待启动 → 进行中 → 已完工 → 已结算 → 已归档
  ↓         ↓         ↓
 已取消   已暂停    已暂停
```

**状态转换条件**：
- 待启动 → 进行中：合同签订完成，项目启动会召开
- 进行中 → 已完工：竣工验收通过
- 已完工 → 已结算：尾款支付完成
- 已结算 → 已归档：质保卡发放，资料归档

#### 3.1.2 项目暂停规则

**暂停条件**：
- 业主申请暂停
- 资金不足暂停
- 重大问题暂停
- 不可抗力暂停

**暂停处理**：
- 记录暂停原因
- 通知所有参与方
- 冻结相关任务
- 保存现场状态

**恢复条件**：
- 暂停原因解决
- 业主确认恢复
- 资金到位

---

### 3.2 任务管理规则

#### 3.2.1 任务状态流转规则

```
待分配 → 待执行 → 执行中 → 待验收 → 已验收
  ↓         ↓         ↓         ↓
已取消   已暂停   已暂停   整改中
```

**状态转换条件**：
- 待分配 → 待执行：任务分配完成
- 待执行 → 执行中：任务开始执行
- 执行中 → 待验收：任务执行完成
- 待验收 → 已验收：验收通过

#### 3.2.2 任务依赖规则

**依赖类型**：
- 完成-开始（FS）：前置任务完成后，后置任务才能开始
- 开始-开始（SS）：前置任务开始后，后置任务才能开始
- 完成-完成（FF）：前置任务完成后，后置任务才能完成
- 开始-完成（SF）：前置任务开始后，后置任务才能完成

**依赖检查**：
- 任务开始前检查前置任务状态
- 前置任务未完成，后置任务不可开始
- 循环依赖检测

---

### 3.3 验收管理规则

#### 3.3.1 验收流程规则

**验收申请**：
- 任务执行完成
- 自检合格
- 提交验收申请
- 上传验收材料

**验收执行**：
- 监理现场验收
- 检查验收标准
- 填写验收单
- 拍照记录

**验收结果**：
- 通过：进入下一环节
- 不通过：整改后重新验收
- 部分通过：部分整改后验收

#### 3.3.2 验收标准规则

**水电验收标准**：
- 水管打压测试合格
- 电路通断测试合格
- 管线走向规范
- 接头处理牢固

**泥瓦验收标准**：
- 墙面平整度≤3mm
- 瓷砖空鼓率≤5%
- 防水试验合格
- 阴阳角方正

**木工验收标准**：
- 尺寸误差≤2mm
- 接缝严密
- 表面光滑
- 结构牢固

**油漆验收标准**：
- 墙面平整度≤2mm
- 颜色均匀一致
- 无流挂、起泡
- 边角顺直

---

### 3.4 资金管理规则

#### 3.4.1 付款节点规则

**付款节点设置**：
```
节点1：合同签订 → 20%
节点2：水电验收 → 30%
节点3：泥瓦验收 → 20%
节点4：木工验收 → 15%
节点5：竣工验收 → 15%
```

**付款条件**：
- 验收通过
- 业主确认
- 资金充足

**付款流程**：
```
验收通过 → 系统触发付款 → 业主确认 → 资金划转 → 到账通知
```

#### 3.4.2 退款规则

**退款条件**：
- 项目取消
- 合同解除
- 超额支付
- 服务未完成

**退款流程**：
```
退款申请 → 平台审核 → 资金解冻 → 退款执行 → 到账通知
```

**退款时效**：
- 审核时效：3个工作日
- 退款时效：5-7个工作日
